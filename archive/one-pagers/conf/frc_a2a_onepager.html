Structured controls for agent provenance, instrumentation trace, and compound verdicts. Agent workflows can be operationally robust but still opaque in decision provenance. FRC A2A adds the traceability layer that turns "agent is attested" into an auditable, reproducible compliance record.

The Problem
  • "Agent is attested" does not guarantee attributable data path
  • Tool calls and connector access are often unlogged at decision level
  • Subagent handoffs create accountability gaps in multi-step flows
  • Mode B (agent-mediated) lacks structured traceability controls
  • Human-in-the-loop assertion is not formally recorded
What FRC A2A Adds
  • L0/L0-D reason codes for agent and data-path risk signals
  • instrumentation_trace — tool calls, connectors, handoffs
  • Mode A vs Mode B submission semantics — formally distinguished
  • Compound verdict = document authenticity × agent provenance
  • Optional agent_verdict enum for agent-layer only assessment
  • Policy profile presets (strict / balanced) for deployment
Submission Modes
Mode A — Human Submission
Document submitted directly by a human via web or mobile interface. Standard KYC onboarding flow.
Required: FRC document layer
Optional: L0 agent context if headers present
Mode B — Agent Submission
Document submitted programmatically by an AI agent (e.g. Anthropic KYC Screener, agentic pipeline).
Required: FRC document layer + full instrumentation_trace
Required: L0-D codes for data path verification
Compound Verdict Logic
Document verdict Agent state Compound verdict Rationale
GENUINE ATTESTED + trace present TRUSTED Full signal chain verified
GENUINE UNATTESTED or trace absent ESCALATE Document real; provenance gap
SYNTHETIC / EDITED / SCREENSHOT Any agent state BLOCK Document failure — agent state irrelevant
INSUFFICIENT ATTESTED REVIEW Insufficient evidence; attested agent
Key L0-D Reason Codes (Data Path Layer)
FRC-L0-DATA-PATH-UNATTRIBUTED
Decision cannot be traced to a verified connector or data source.
FRC-L0-CONNECTOR-OUT-OF-POLICY
Data source accessed outside the declared connector allowlist.
FRC-L0-HANDOFF-UNAUDITED
Subagent handoff has no paired log entry in instrumentation trace.
FRC-L0-TOOL-PERMISSION-VIOLATION
Tool invoked outside its declared permission scope.
FRC-L0-SECRET-BINDING-UNKNOWN
Credential binding to workload cannot be confirmed.
FRC-L0-HITL-ASSERTION-MISSING
No recorded human confirmation before high-risk action in flow.
Key Metrics
ABR
Agent Bypass Rate
% agent submissions with anomalies receiving TRUSTED verdict.
ACG
Agent Confidence Gap
Mean confidence on approvals with L0 flags present.
CDR
Chain Disruption Rate
% submissions with broken delegation chain.
TCR
Tool Call Coverage
% submissions where all external calls have valid connector_id.
HAR
Handoff Audit Rate
% multi-agent runs where every handoff has a paired log.
